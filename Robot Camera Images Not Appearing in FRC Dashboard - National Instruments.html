    Robot Camera Images Not Appearing in FRC Dashboard

    Updated Mar 7, 2019

    Reported In

    Reported In shows products that are verified to work for the solution described in this article. This solution might also apply to other similar products or applications.

    Hardware

    • roboRIO-FRC

    Software

    • FRC 2019 Software

    Issue Details

    I'm working with a camera on my FIRST Robotics Competition robot, but I'm not seeing any images coming from my roboRIO to my dashboard. What troubleshooting steps should I try?

    Solution

    There are many potential causes for webcam difficulty, but the following steps will help narrow down where the issue is happening and resolve the most common cases:
      
    1. Make sure you have the most recent version of the NI Update Suite installed on all your computers (your development machine(s), as well as the computer you use during matches).
    2. Use the newest version of the Dashboard.
      1. If you're using the default Dashboard:
        1. Run the 2019.2.1 dashboard, which is attached to this document. It includes a longer timeout for camera connections to the field.
      2. If you are using a custom LabVIEW Dashboard:
        1. Close all applications.
        2. Unzip the file attached to this document and copy the vi.lib folder over the C:\Program Files (x86)\National Instruments\LabVIEW 2018\vi.lib folder to merge the modified functions with the rest of the LabVIEW's vi.lib functions.
        3. Open your Dashboard project in LabVIEW to reload the updated VIs.
        4. To add a way to check the robot's IP, use the path C:\Program Files (x86)\National Instruments\LabVIEW 2018\vi.lib\Rock Robotics\WPI\Dashboard to open a File Explorer window, and drag/drop the file WPI_DashboardGet Robot IP.vi directly to the Dashboard Main Block Diagram. Wire the required input, called the Notification Refnum, as shown below. 

        5. Build your dashboard from the Build Specification, and navigate to where you saved the new executable file. Copy the new Dashboard.exe file over to the C:\Program Files (x86)\FRC Dashboard directory, replacing the existing one.
    3. Make sure the roboRIO has been imaged with the latest update. This will resolve any issues caused by missing files. This can also help resolve Error 7 or 1003 in the Driver Station logs.

    4. Check your Robot Main.vi file. It should include both of the VIs highlighted in the image below.

      1. Close all applications.
      2. Unzip the file attached to this document and copy the vi.lib folder over the C:\Program Files (x86)\National Instruments\LabVIEW 2018\vi.lib folder to merge the modified functions with the rest of the LabVIEW's vi.lib functions.
      3. No other code is needed to send the images from a USB camera to the Dashboard with LabVIEW.
      4. To verify whether coding mistakes may be part of the issue, make sure you're testing with a default dashboard and a default robot project to rule out any potential programming errors affecting your results.
    5. Build your robot code and set it to Run as Startup, as shown in the images below.

    6. Check that the Dashboard sees the Network Tables correctly. If you have difficulty with this step, make sure your firewalls are off, you've instructed Windows to allow the app, and your third-party antivirus is disabled. You can also try restarting the Driver Station application and your Dashboard.

    7. Select the camera from the Dashboard’s menu. If you do not see the camera in the list, or the camera display toggles between robot addresses without counting down or showing the name of the camera: 
      1. Try restarting the robot code using the button highlighted below:

      2. Make sure the roboRIO can see the camera correctly. You can do this using NI MAX. Find your roboRIO under the Remote Devices section on the left. When you expand the listing for your roboRIO, you should see your camera. 
      3. Determine if the camera is communicating with the roboRIO by running the Camera Compatibility Test tool on your roboRIO. This will connect to your camera and run through all of the different configuration options.
        1. While the test is running watch the stream on the left to make sure the camera feed is coming through.
        2. When the test is done, check if any modes are listed as successful. Try setting your camera to one of these modes.
    8. If the camera is detected by the roboRIO and can be seen in the Dashboard’s drop-down menu, but the image is still not displaying, then try lowering the resolution and increasing the compression.

      1. If none of the modes are successful and you don't see any images on the camera stream, the camera may not be working correctly. Plug the camera directly into your PC to verify that the hardware works.
        1. If you have LabVIEW installed on your computer, you have access to a tool called NI MAX. This tool will not only allow you to test whether your camera captures images, but also allows you to manually adjust the camera's settings for more in-depth testing. See Additional Information below for information about how to use NI MAX to test your camera.
        2. If you do not have LabVIEW installed, you can use any programs installed on your PC that use a webcam (like Skype, Debut, Windows Camera, etc) to test.
    9. Try with an additional laptop or roboRIO to isolate if the behavior is device specific or not.

    Additional Information

    You can find more information about creating and building a dashboard here: FRC Dashboard Tutorial.

    Best practices to use with the Field Management System (FMS) during a match:

    1. Make sure all applications on the laptop are closed.
    2. Place the robot on the field and power it up.
    3. Connect the Driver Station’s Ethernet cable to your laptop.
    4. Open the FRC Driver Station executable.
    5. The Driver Station should bring up the FRC Dashboard, too. 
      1. Open the FRC Dashboard if this is not the case.
    6. Select the camera in the dashboard (if not selected already).
    7. The image should appear shortly after communications are established.
      1. If you do not receive an image, go through the troubleshooting process described in this article.
     

    Using NI MAX to test your camera

    If you have LabVIEW installed, you can test your camera using the NI Measurement and Automation Explorer, or NI MAX. Here's how to access your camera from within NI MAX:
    1. Open NI MAX, and under My System, expand Devices and Interfaces. Click on your camera in the list.
    1. Click one of the acquisition options along the top of the window, to determine whether your camera hardware is working as expected.
      • Snap will take one test image
      • Grab will start streaming test images until clicked a second time

    Note: If you would like to access specific camera properties, the most common ones will be shown on the Acquisition Attributes page. The rest are available under the Camera Attributes tab.
     

    2019.2.1 Release notes

    FRC Dashboard

    The LabVIEW/default dashboard implements a retry timeout and will close the connection if a valid image is not received after N seconds -- possibly resulting in an error 66 being reported from the roboRIO LabVIEW code. In the 2019 kickoff version of the dashboard, that timeout was set to 10 seconds. For the 2019.2 and 2019.2.1 versions, it has been extended to 30 seconds. The increased timeout allows for longer connection time.
     
    The dashboard also relies on a localhost socket to be told the robot IP address to establish a Network Tables connection. In some situations, this socket doesn't connect, leaving the dashboard running without a connection to Network Tables. This may also impact camera connection since camera descriptions are shared via Network Tables. The 2019.2.1 dashboard uses an additional socket and protocol for receiving the robot IP address from the Driver Station. Even with this fix, we still see some situations where the Driver Station knows the robot IP, but the dashboard doesn't connect network tables. If this occurs, restart the dashboard and possibly the driver station.
     

    Robot

    The LabVIEW camera server, implemented in WPI_CameraSend Images To PC Loop.vi, is responsible for publishing the URLs that enable the dashboard to connect to cameras connected to the roboRIO. The list of URLs include both hostname and resolved IP formats. If the camera server code executes and resolves the IP before the robot radio has assigned an IP address, this can result in an invalid IP address being latched and published. Additionally, if the connection to robot changes connection (for example: moving from USB connection to WiFi), the published IP address may not work for the new network. Ideally, the hostname form of URL would resolve, allowing for successful camera connection, but current FMS networking often fails to resolve successfully.
     
    The code to publish the URLs for the camera server now includes all listed interfaces except for localhost forms. It will also republish about every 5 seconds in case the network type changes or completes booting. The host name form is still included in the list of URLs.
